# Standalone HTML Audit Reports

This document explains the architecture, design choices, safety patterns, and UX decisions for OpenGrowth's standalone HTML report generator.

## 1. Standalone / Offline-First Design

The HTML report generated by OpenGrowth (`opengrowth-report/report.html`) is designed to be **completely standalone** and **100% offline-ready**.

* **Self-Contained Styling:** All layout and component styles are embedded in a `<style>` block inside the `<head>`. There are no external stylesheets, ensuring the document renders identical to design specs even in low-connectivity/offline environments.
* **No CDNs or External Dependencies:** All fonts, icons, styling frameworks, and analytics are omitted. Standard system fonts (`system-ui, -apple-system, Segoe UI, Roboto, Helvetica, Arial, sans-serif`) are used to render clean, readable typography with no loading latency.
* **Pure HTML Progressive UX:** Instead of relying on JavaScript libraries for accordion sections or tabs, the report uses semantic HTML `<details>` and `<summary>` tags for expandable check details. This avoids execution latency, eliminates JS execution security risks, and keeps the document lightweight.

## 2. Included Sections

The HTML report contains 19 core sections ordered logically for an agency-grade audit walkthrough:

1. **Hero / Summary:** Basic page details, final audit score, rules count, and high findings counter.
2. **Scorecard:** Circle score gauge and overall passed rule stats.
3. **Category Scores:** Category-level scorebars showing relative scores.
4. **High Priority Findings:** Any failed rules flagged with `critical` or `high` severity.
5. **Website Intelligence:** Inventory of crawled metadata (title, description, links, canonical, H1, etc.).
6. **Rule Results:** Expandable detailed checklist of all rules evaluated, grouped by category.
7. **Content Strategy:** Strategic content overview and extracted core keywords.
8. **Topic Clusters:** Structured keywords clustered by strategic user search intent.
9. **Content Gaps:** Detected content holes mapped to high-conversion recommendations.
10. **30-Day Content Calendar:** Day-by-day organic posting schedule.
11. **Ad Strategy:** Calculated user acquisition segments and value proposition pillars.
12. **Audience Segments:** Detailed buyer persona breakdowns (pain point, outcome, suggestions).
13. **Ad Hooks:** Copy hooks categorized by Hook Family and Platform Fit.
14. **Ad Copy Variants:** Structured copy configurations (ToFu/MoFu/BoFu) across platforms.
15. **Short Video Concepts:** Short-form video frameworks and scripts.
16. **Carousel Concepts:** Visual slide-by-slide concepts.
17. **Creative Directions:** Visual theme directions (design cues, imagery directions).
18. **Recommended Next Steps:** Clear prioritize action item checklist.
19. **Version Notice:** Generation metadata and CLI package release version tag.

## 3. Escaping & Safety Model

Security is a primary requirement. Since OpenGrowth crawls raw page contents and parses user-provided business context, all string outputs injected into the HTML must be treated as untrusted.

### XSS Prevention
We implement robust escaping via `escapeHtml(value: unknown)`:
* `&` -> `&amp;`
* `<` -> `&lt;`
* `>` -> `&gt;`
* `"` -> `&quot;`
* `'` -> `&#39;`

Every user context string, page title, heading string, rule evidence value, and strategy description is passed through `escapeHtml` immediately before insertion. Malicious injection vectors such as `<script>alert(1)</script>` are safely rendered as plain text in the browser.

## 4. UX & Design Goals

* **Agency Quality:** Polished slate (`#0f172a`), emerald (`#10b981`), and amber palette to convey professional analysis.
* **Scannability:** Clear severity chips (`critical`, `high`, `medium`) and visual progress bars to help founders scan outcomes in 5 seconds.
* **Print Friendly:** The CSS includes simple typography scaling and layout margins optimized for PDF printing or sharing.

## 5. Limitations & Future Direction

* **Dynamic Filters:** Lacking JS, filtering by tag or searching rules interactively is not supported. Advanced interactivity will be deferred to the planned **OpenGrowth Dashboard**.
* **Chart Complexity:** Complex graphs are avoided to maintain local-first speed. In a future release, a full dashboard or CLI-guided local server will handle interactive charts and dynamic filtering.
