# OpenGrowth Report Gallery

The Report Gallery is a curated directory of OpenGrowth audits, content calendars, and ad angle blueprints for real and fictional websites. It serves as an inspiration and benchmark database for founders, markers, developers, and copywriters.

## Curated Gallery Roadmap

In the future, this directory will host complete HTML, Markdown, and JSON audits for various business categories:
* **B2B SaaS**: Focus on feature description clarity, self-serve funnels, and product value propositions.
* **E-Commerce**: Focus on trust signals, product details schema, and retargeting hooks.
* **Developer Tools**: Focus on copy clarity, documentation access, and command-line code snippet readability.
* **Digital Agencies**: Focus on client onboarding paths, social proof, and case studies.

## Contributing an Example Report

We welcome contributions of growth reports to help others learn! To add a report:

1. **Anonymise Sensitive Data**:
   * **URLs**: If you audited a private client or pre-launch site, replace all domain references with fictional ones (e.g. `clientdomain.com` -> `mysaastool.com`).
   * **Identifiers**: Anonymise any usernames, emails, tokens, or custom branding that your client does not want public.
2. **Include the Full Asset Output**:
   * Export the 9 standard reports generated by `opengrowth audit <url>`.
   * Keep file sizes small by removing any bloated HTML files or images.
3. **Submit a Pull Request**:
   * Create a folder under `examples/report-gallery/` named after the business type (e.g. `examples/report-gallery/b2b-saas-demo/`).
   * Place the 9 files inside it.
   * Add a short description to the gallery index.

## Submission Guidelines & Rules

> [!WARNING]
> * **No Private Content**: Never submit reports for private repositories, internal staging portals, or password-protected client staging sites unless you have explicit written permission from the owner.
> * **Anonymization first**: If in doubt, anonymise. We reserve the right to request changes or reject PRs containing identifiable client data.
